# Report information architecture

Use when **`deliverable` includes `report`** (executive summaries, KPI packs, narratives with figures).

## Required outline pattern

1. **Title + metadata**: date/version/audience/confidentiality cues (non-secret).
2. **Executive snapshot**: 3 bullets max answering "so what?"
3. **Key metrics**: tight band of charts/tables referencing the same timeframe.
4. **Findings**: ordered sections aligned to questions or dimensions (not tooling).
5. **Recommendations / next steps**: decisions with owners placeholders when unknown.
6. **Appendices**: methods, glossary, definitions, extended tables.
7. **Sources / links**: static footnote block or appendix.

## Typography & markdown rules

- One `h1`; section `h2`; sub-points `h3`. Never skip ranks.
- Long lists break into chunked cards with captions.
- Code blocks fenced and labeled (` ```ts ` naming optional ).
- Figures use `<figure>/<figcaption>`; reference in prose before the figure appears.

## Density control

Executive layer ≤ ~400 words visible before scrolling on desktop; deepen below fold. Repeated metrics share one scaffold to avoid contradictory filters.
