# Authoring skill evals (jstack.core)

## Layout

- Each skill may have a `skills/<path>/evals/` folder with YAML case files (see `evals/discover.ts`).
- Repo-wide skill index: `evals/evals.json` (list of skill paths).

## Smoke and routing quality

- **Smoke (`001-skill-smoke.yaml`)** — Hypothetical outline; no fake API results.
- **Negative (`002-negative-trivia.yaml`)** — Short control case.
- **Graded (`003-graded-assert.yaml`)** — Rubric + light asserts when present.

### Orchestrators

Domain orchestrators (`jira`, `knowledge`, `session`, `notion`, `meetings`, `research`, `reports`, `self`, `review`, `metrics`, `routines`, `workflows`, `incident`, `sop`, `sprint`) should have **at least two different wordings** that expect the same routing behavior:

1. The standard smoke case.
2. **`004-paraphrase-routing.yaml`** — Generated by `bun run scripts/generate-skill-evals.ts` when missing; you may hand-edit the embedded user scenario to match your team’s language.

When adding triggers in `apply_detailed_skills_data.py` (`DESCRIPTIONS`, `WHEN_TO_USE`), add or adjust a **paraphrase** line in `004-paraphrase-routing.yaml` so evals stay aligned with discovery strings.

## Commands

- Scaffold / refresh eval files: `bun run scripts/generate-skill-evals.ts` (from `jstack.core`, or via package script if wired).
- Use `--dry-run` to preview paths only.
- **CI:** Standalone `jstack.core` repo — `.github/workflows/skill-eval.yml` (path-scoped `eval:validate` + `eval:quick`). Monorepo root — `.github/workflows/jstack-core-skill-eval.yml` runs the same under `jstack.core/`. LLM semantic evals stay local (`bun run eval:semantic`).

## Scenario packs (cross-skill stress)

- Packs live under `evals/scenarios/packs/*.yaml`. Each pack defines `scenarios[]` (prompt, criteria, optional `assert`, `files`, `strict_grader`) and `default_targets` (skill rel paths).
- **Validate YAML (no API):** `bun run eval:scenarios-validate`
- **Run (Claude CLI execute + judge, same pipeline as per-skill semantic):** `bun run eval:scenarios [--pack core-stress] [--skill recon] [--all-skills] [--max-skills N] [--threshold 72] [--viewer]`
- **Plan without API:** `bun run eval:scenarios --dry-run` — lists expanded case names per skill and validates case shape.
- **Templates:** `{{skill_path}}` / `{{SKILL_PATH}}` in prompts and criteria are replaced with the target skill rel.
- **Strict grader:** set `strict_grader_default: true` on the pack or `strict_grader: true` on a scenario, or set env `JSTACK_EVAL_STRICT_GRADER=1` (see `evals/grade.ts`).
- **Pass threshold (scenarios):** `--threshold` if set, else pack `default_threshold` if set, else per-skill `evals/eval-config.yaml` `pass_threshold`, else `JSTACK_EVAL_PASS_THRESHOLD`.
